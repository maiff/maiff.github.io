一个面试题引发的思考

发表于 2016-02-22   |   分类于 JS的世界   |  

在微信公众号上看到一个面试题
大概内容是这样

答案

原链接都有自己的解释，我来说说我引发的理解和尝试。

  • 第一个访问属性没问题。

  • 第二个就有问题了，按照原题主的解释，涉及到函数表达式和函数声明的区别，还有变量声明要提前的问题。

    var x=1;
function x(){}
alert(x);//1

alert(x);//function x
var x=1;
function x(){}

    因为变量声明要提前，var x会被提到最前面，函数声明也会，然后赋值语句，在某些情况下会覆盖声明，如代码中所示，var x=1被分解为var x和x=1，var x被提到最前，函数声明也被提到最前然后再到赋值语句，所以被覆盖了。在原题中全局变量“alert（4）”覆盖掉“alert（5）”

    • 相关链接1-揭秘命名函数表达式
    • 相关链接2-函数声明 VS 函数表达式
  • 第三个我在理解的时候又出现问题了，没弄懂那个return this返回的是谁，后来知道是window，由于那个函数没有var，所以污染了全局变量，覆盖了所以弹出为1。
  • 同理第四个也是。
  • 第五个，原题主说是优先级的问题，可以看成new (Foo.getName)()所以是2
  • 第六个，可以看成(new Foo()).getName()，一般new的对象是没有返回值，有返回值要就情况考虑，这里就是Foo的实例，因为没有为实例添加任何属性所以（在这里我还有别的理解后来知道this.attr，跟Foo.attr不一样），会向原型上查找，找到所以是3.
  • 第七个可以看成new((new Foo()).getName)()，就是原型上的getName的构造函数所以返回的是3.

总结：
看着很烦，这要是我一开始就看这个肯定看不懂，而且会感觉到很烦，但是现在有比较稳健的基础，看了之后就有种豁然开朗的感觉，虽然这篇短博客我写了前前后后查资料花了我超过一个小时的时间，要静下心来就有收获。
