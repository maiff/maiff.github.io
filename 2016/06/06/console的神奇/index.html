console的神奇

发表于 2016-06-06   |   分类于 JS的世界   |  

最近看了一篇张旭鑫老师的文章，原来以为自己不浮躁，还是举得自己很浮躁。
文章链接
我以为我自己到瓶颈了，我以为我自己很厉害了，其实不然，我以前的学习只是扩大知识面但并没有掺杂很多思考，学而不思则罔，所以我之前才会迷茫，从现在开始新知识的吸收要做到下面几个部分：

  • 扩展知识点
  • 对比知识点
  • 深入本质
  • 系统的学
    不要认为这样的学习很慢，其实不然。

好了废话多说今天要介绍的主角是console。我用代码输出了console的属性吓我一跳，在chrome上有25个，在火狐上有22个，

不一样的地方

  • chrome有memory、markTimeline、timeline、timelineEnd
  • 火狐则没有这四个但也有chrome没有的exception

先说说这几个不同的属性，为什么不同，因为不同的标准不同的兼容性，memory会显示
内部栈的使用情况。
markTimeline则被废弃被timeStamp所代替，这个属性ie不兼容，然后我尝试在chrome中使用并没有反应什么都没有打印，在火狐中打印时间字符串，然后我就想怎么用在stackflow上看到，会在timeline上建立一个时间戳，而timeline也被time所代替，然后exception跟error一样。
说完不一样，我来说说一样的。

time()、timeEnd()

一开始我测试一段代码，都很傻的生成一个时间，在结束的的时候在生成一个时间，再减一下，然后今天才知道可以这样，用console.time()和console.timeEnd()

log()，info()，debug() ,warn() ,error()

我今天才知道log()还能像c的printf那么用，不过有点差别

  • %s 字符串
  • %d 整数
  • %i 整数
  • %f 浮点数
  • %o 对象的链接
  • %c CSS格式字符串
    特别是%c可以让log有样式。
    info、debug、warn、error都差不多，只是显示的不一样。

table()

可以以表格的显示出来，

var animals = [
    { animal: 'Horse', name: 'Henry', age: 43 },
    { animal: 'Dog', name: 'Fred', age: 13 },
    { animal: 'Cat', name: 'Frodo' }
];

console.table(animals);

count()

这个可以计算一个函数被调用了多少次。

function greet(user) {
  console.count();
  return "hi " + user;
}

greet('bob')
//  : 1
// "hi bob"

greet('alice')
//  : 2
// "hi alice"

greet('bob')
//  : 3
// "hi bob"

还可以分类

function greet(user) {
  console.count(user);
  return "hi " + user;
}

greet('bob')
// bob: 1
// "hi bob"

greet('alice')
// alice: 1
// "hi alice"

greet('bob')
// bob: 2
// "hi bob"

好了其实上面大部分都是抄阮一峰老师的博客在这我没说的都在上面。

console对象在node上有多少

因为最近在看node不禁想起console在node上有多少好用的都在上面，一些格式性的显示不出来也就没有了。 同时我注意多了一个Console属性，应该是构造器属性。

总结

虽然这前前后花了我很多时间，来看这个console，但是我相信大部分人没有像我这么了解，以后做学问也要这样，突破瓶颈，不能半瓶醋。
