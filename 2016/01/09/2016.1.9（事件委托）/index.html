发表于 2016-01-09   |   分类于 JS的世界   |  

2016.1.9

今天不知道为什么好开森。
今天看起来码了很多代码其实根本就没有多少。
总结一下：

  • 事件绑定
    • 兼容
      element.addEventListener(event, listener, false);
      element.attachEvent("on" + event, listener);
      element["on" + event] = listener;
      PS.这里对第三个参数进行说明,它会改变事件执行顺序，是冒泡还是捕获。
    • 解绑的兼容
      element.removeEventListener(event, listener, false);
      lement.detachEvent("on" + event, listener);
      element["on" + event] = null;
  • 事件代理/委托
    • 基础就是事件冒泡

为父节点添加一个click事件，当子节点被点击的时候，click事件会从子节点开始向上冒泡。父节点捕获到事件之后，通过判断e.target.nodeName来判断是否为我们需要处理的节点。并且通过e.target拿到了被点击的Li节点。从而可以获取到相应的信息，并作处理。

对应代码：

 element.addEventListener(eventName, function(ev){
    var oEvent=ev||event;
    var target=oEvent.target||oEvent.srcElement;
    if(target.nodeName.toLowerCase()==tag){
        listener(oEvent);
    }
}, false);

  • JS中对cookie的操作

    function setCookie(cookieName, cookieValue, expiredays) {
    // your implement
    var nowdate=new Date();
    nowdate.setDate(nowdate.getDate()+expiredays);
    document.cookie=cookieName+'='+escape(cookieValue)+((expiredays==null)?'':';expires='+nowdate.toGMTString())
}

function getCookie(cookieName){
    if(document.cookie.length>0){
        var cookie_start=document.cookie.indexOf(cookieName+'=');
        if(cookie_start!=-1){
            cookie_start=cookie_start+cookieName.length+1;
            var cookie_end=document.cookie.indexOf(';',cookie_start);
            if(cookie_end==-1)cookie_end=document.cookie.length;
            return unescape(document.cookie.substring(cookie_start,cookie_end));
        }
    }
}

PS.对于value进行编码在mozilla的JS的MDN中escape已经废弃就没有用了
