ife设计模式与细节知识的思考

发表于 2016-03-23   |   分类于 技术应用   |  

ife收获

CSS做的实在做不下去了，各种盒子模型，各种定位方式。实在受不了。然后我就开始做JS的项目，发现还是蛮简单的，但我从简单中学到很多。
比如一种设计模式，姑且叫设计模式吧。

demo地址

这个是根据选择不同的表格时间颗粒度，和不同城市，来呈现不同的柱状图。
这个设计模式是怎样的，选择不同的状态把这不同的状态存到相应的对象里，再根据这些对象里的数据再创建一个对象放入渲染DOM节点所需要的数据。然后用一个函数取到数据来渲染DOM节点。
大概思路就是这样不过要看代码才能理解。

生成随机颜色

参考链接

  • 司徒正美

按照我的思路是可以准备一个候选颜色，存入数组，然后随机选择，但是如果数据量太多，就会出现很多想同同时，存入会影响文件的大小。

方法一

var getRandomColor = function(){
  return  '#' +
    (function(color){
    return (color +=  '0123456789abcdef'[Math.floor(Math.random()*16)])
      && (color.length == 6) ?  color : arguments.callee(color);
  })('');
}

三元式加递归，加闭包，生成六个随机字符。

var getRandomColor = function(){
  return (function(m,s,c){
    return (c ? arguments.callee(m,s,c-1) : '#') +
      s[m.floor(m.random() * 16)]
  })(Math,'0123456789abcdef',5)
}

用c的状态来判断是否调用自身

var getRandomColor = function(){
  return '#'+Math.floor(Math.random()*16777215).toString(16);
}

16777215是十六进制最大的数转化为十进制，位数不足。但浏览器兼容性很好。

var getRandomColor = function(){
  return '#'+(Math.random()*0xffffff<<0).toString(16);
}
