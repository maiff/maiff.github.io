发表于 2016-03-25   |   分类于 再理解学习   |  

HTML的再学习

<area>标签，代码实例
<aside>标签:就是一个标签代表额外内容
<base>标签：设置一个基准url和基准目标，详见
<bdi>标签把文本从父元素中隔离出来
<bdo>像镜子一样
<blockquote>长引用
<button>重要事项：如果在 HTML 表单中使用 button 元素，不同的浏览器会提交不同的值。Internet Explorer 将提交<button> 与 <button/>之间的文本，而其他浏览器将提交 value 属性的内容。请在 HTML 表单中使用 input 元素来创建按钮。
请始终为按钮规定 type 属性。Internet Explorer 的默认类型是 “button”，而其他浏览器中（包括 W3C 规范）的默认值是 “submit”。
注意那些form属性
<caption>表格标题
<col>亲测align valign没有用 width span都可以。
Firefox、Chrome 以及 Safari 仅支持 colgroup 元素的 span 和 width 属性。
<colgroup>感觉跟col没区别。
<datalist>配合input使用用list绑定datalist。
<datails>summary是标题，可以用做导航，只有chrome和safari支持
<dfn>定义定义项目
<dialog>只有chrome和safari支持，会话窗口
<embed>定义嵌入的内容
<fieldset>标签<legend>是这个标题
<figure>定义媒体分组可以用figcaption设置标题
<frameset>定义frame集合
<iframe>放着功能很强大
<img>图像映射
<input>有很多HTML5的属性很哟用
<keygen>定义生成秘钥
<mark>定义有记号的文本
<meter>定义给定度量范围IE不支持
<object>定义一个对象好像没用……
<optgroup>给select分组
<output>输出IE不支持
<progress>进度
<ruby><rt><rp>定义ruby注释同左若不支持ruby
<sub><sup>下标上标
<wbr>防止在错误的地方换行

属性

accesskey:Alt + accessKey (或者 Shift + Alt + accessKey)所有浏览器都支持。
contenteditable:元素是否可编辑，所有浏览器都支持
data-* 属性来嵌入自定义数据，兼容性同上
draggable：需要js配合，兼容性同上
hidden：hidden 属性是布尔属性。如果设置该属性，它规定元素仍未或不再相关。浏览器不应显示已规定 hidden 属性的元素。兼容性同上
spellcheck：拼写检查，兼容性同上
translate：是否被翻译，兼容性同上
tabindex：tab的次序，兼容性除safri

1.今天对HTML的学习，更加具体系统，同时我感觉到需要规范，同时HTML5更偏向规范，更偏向语义化
